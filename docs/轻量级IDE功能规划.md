> 基于 Claude Code GUI 现状的 IDE 化改造分析  
> 后端（编码、AI）由 claudecode CLI 完成，前端聚焦文件操作、项目导航、上下文可视化

## 二、IDE 化缺失功能清单

按「价值 / 工作量」分级排序。**编号连续，便于对照实现与评审。**

### 🔥 P0 — 核心 IDE 体验缺失

#### 1. 文件浏览器（File Explorer）

- **现状**：侧边栏「项目」Tab 只有项目→会话树，看不到仓库内文件
- **建议**：在同一侧栏 Tab 栏增加 **「文件」**（与「技能」「项目」并列），树形展示**当前会话关联项目目录**为根路径
- **价值**：IDE 最基本导航；可选「引用路径 / @ 文件」送入上下文
- **形式**：复用现有侧栏 Tab 切换模式，不必另起停靠窗口
- **注意**：**单一事实来源**——文件树根目录应与 `ProjectManager` 中当前项目路径一致，而非与已折叠工具栏内遗留 `WorkingDirBox` 各行其是

#### 2. 代码编辑器 / 查看器（Code Viewer）

- **现状**：代码主要在聊天气泡中查看，难以独立编辑、保存
- **建议**：点击文件树或消息内路径后，在 **右侧或底部可关闭面板** 打开视图；阶段目标：**只读预览 → 可编辑保存**
- **价值**：改完后就地核对最终文件内容的最短路径
- **形式**：短期可 RichTextBox + 现有高亮；若编辑体验不足，规划升级 **AvalonEdit**（WPF 常用）或阶段性「外链系统编辑器」
- **策略**：大文件 / 二进制——阈值以上仅显示「路径 + 大小 + 可选摘录」，避免拖垮 UI；写入前再考虑冲突提示

#### 3. Diff 视图（Diff Viewer）

- **现状**：FileSummary 仅 +N / -N，无语义级变更阅读
- **建议**：点击摘要行或 ToolCall 关联路径展开 **行级 diff**（增删着色，与主题 `ThemeDiffAdded/Removed` 一致）
- **价值**：AI 改动的核心可读性
- **形式**：内联可折叠块或独立浮动窗口二选一即可起步

#### 4. 状态栏增强（Status Bar）

- **现状澄清**：**已有**底栏 `StatusBar` + `TurnCounter`，但属于「单行消息型」，不是 IDE 分区栏；ThinkingBar 只管会话内进度与任务
- **建议**：在同一底栏row内划分为 **左 / 中 / 右**（可用多列 `TextBlock`）：例如 **项目短路径或文件夹名**、**Git 分支 + 变更数（占位可先 `-`）**、**当前模型名 + Build/Plan 模式**；保留原有「就绪 / 错误」类动态消息时可采用「左侧主消息 + 右侧固定信息」叠放策略
- **价值**：持续感知工作上下文
- **技术债**：底栏容器当前存在硬编码背景色（如 `#141416`），IDE 化时应改为 **DynamicResource**，与文档第四节原则一致

#### 5. 快捷键体系（含快速打开）

- **现状**：逻辑散落在 `PreviewKeyDown` / `KeyDown`，无统一命令表
- **建议**：在窗口层集中 **`InputBinding` / `Command`**，并维护一张可文档化的快捷键表；**Ctrl+P** 在本条一并落地（即「快速打开文件」行为，见下表），不再单独占一条编号以免与 P1 重复

| 快捷键 | 功能 |
|--------|------|
| Ctrl+N | 新建会话 |
| Ctrl+W | 关闭当前标签 |
| Ctrl+Tab | 切换标签 |
| Ctrl+P | 快速打开文件（模糊匹配项目内路径） |
| Ctrl+F | 当前会话时间线内搜索 / 跳转（先做聊天内即可） |
| Ctrl+` | 切换输出 / 终端面板（面板存在后绑定） |
| Ctrl+Shift+F | 项目中全文搜索（面板存在后绑定） |
| Ctrl+Shift+P | 命令面板（可与 P2 第 11 项同一实现） |

- **价值**：专业工具分水岭；为后续「设置里改键」留扩展点

### ⚡ P1 — 大幅提升效率

#### 6. 输出 / 终端面板

- **建议**：底部可折叠面板，承接 **构建 / 测试 / shell** 等长文本输出，与聊天时间线解耦
- **价值**：工具输出可读性；必要时从现有子进程 stdout **分流**到面板（未必改 CLI 协议）
- **备注**：若首版仅「只读镜像 stdout」，仍属前端分流，与第四节「不强制改 claudecode 契约」兼容

#### 7. 文件拖放 + 右键集成

- **建议**：文件拖入输入区 → 附加路径或受控读取摘要；资源管理器右键扩展可作为远期；应用内文件树 **右键「发送到 Claude / 引用路径」**
- **价值**：降低手动复制成本

#### 8. 从 ToolCall / FileSummary 跳转到编辑器

- **建议**：点击卡片或摘要中的 **文件路径**，若在项目根下则打开 **第 2 项 Viewer**，并尽量携带 **行号**（若事件中有）
- **价值**：闭环：「说了改了什么」→「立刻看到文件」

---

### 💡 P2 — 锦上添花

#### 9. Git 集成（基础版）

- **建议**：状态栏分支名 + 变更数；点击弹出简略变更列表（后续再接 diff）
- **价值**：高频改动的上下文感知

#### 10. 命令面板（Ctrl+Shift+P）

- **建议**：模糊搜索可执行动作（切换主题、RuntimeMode、打开设置、聚焦文件树等）
- **价值**： discoverability；与 P0 快捷键表同一入口逻辑

#### 11. 搜索面板（Search in Files）

- **建议**：项目内全文搜索，结果跳转 **Viewer** 指定行
- **形式**：侧栏 Tab 或模式浮层均可

#### 12. Markdown 预览

- **建议**：`.md` 双模式：源码 / 预览（技能与 README）

#### 13. 问题 / 错误列表（Problems）

- **建议**：解析编译器、lint、测试失败等 **结构化或可正则的稳定格式**，汇总列表面板
- **价值**：与 **ThinkingBar 任务列表** 区分：后者是「计划步骤」，前者是「客观错误诊断」

#### 14. 分栏布局

- **建议**：聊天区与文件 / 预览区 **可拖拽分栏**（左右或上下）

#### 15. 工程化配套（持久化、设置、面包屑、导出、无障碍）

- **布局持久化**：侧栏宽度、底部面板高度写入 `%APPDATA%`（与 SQLite 会话路径策略一致）
- **设置 / 键位**：集中页或配置文件，避免快捷键写死在多处 handler
- **面包屑 / 标题区**：副标题或标题栏展示「当前项目名 / 会话名」或缩短路径，减轻只看底栏长文案的负担
- **导出会话**：Markdown 等格式导出当前标签对话，便于复盘与提 issue
- **无障碍**：弹出层焦点陷阱、`AutomationProperties`、主要按钮可读名称

## 三、优先级建议（修订）

```
立即能做（P0，高价值 / 相对独立）
  ├── 快捷键体系（窗口级 Command + InputBinding）— 含 Ctrl+P
  ├── 状态栏分区扩展 — 在现有底栏 row 上改列布局；顺手去掉底栏硬编码色
  ├── Diff 视图 — 点击 FileSummary / 工具卡片展开
  └── 文件拖放入口 — 输入区 Drop 可先做起

短期接续（核心 IDE 闭环）
  ├── 侧栏「文件」Tab + 与项目路径单一来源
  ├── 代码 Viewer（先只读后编辑）；大文件策略写清
  ├── ToolCall / 摘要路径 → 打开 Viewer
  └── 会话内 Ctrl+F 搜索

中期
  ├── 输出 / 终端面板（stdout 分流）
  ├── 命令面板（与快捷键共用命令注册表）
  └── Git 状态只读集成

长期（体验抛光）
  ├── 全文搜索面板、Markdown 预览
  ├── Problems 面板、分栏布局
  └── 持久化、设置、导出、a11y 细化
```

## 四、与现有架构的兼容性

改造原则：

1. **不破坏现有聊天核心** — 新能力以独立控件 / 面板 / Command 注册为主；**尽量少侵入** `MainWindow.Session.cs`、`MainWindow.Chat.cs` 的发送与解析主路径，必要时仅增加「事件订阅 / UI 分流」钩子
2. **复用现有主题系统** — 新控件颜色通过 `Theme.cs` + `DynamicResource`；**新增代码禁止引入新的硬编码主题色**（含底栏背景）
3. **增量开发** — 每项尽量可独立交付；文件树不依赖 Diff，Diff 可先对接已有摘要数据
4. **CLI 契约** — 默认 **不修改** claudecode 调用协议与参数规范；终端面板、问题列表等属于 **前端展示层增强**：可读现有流、可选附加解析，若将来需要 CLI 侧 JSON 流再单独立项评估
